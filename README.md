# Лабораторная работа: Электронный журнал (Web-based Gradebook)

Вы знаете как я его ненавижу)

## Цель работы

Освоить проектирование и разработку современного веб-приложения для автоматизации учёта посещаемости, оценок и взаимодействия между студентами и преподавателями. Получить практические навыки работы с современными фреймворками для фронтенда и бэкенда, а также с REST API, валидацией, авторизацией и хранением файлов.

## Задание

Разработать электронный журнал с поддержкой двух ролей: **Студент** и **Преподаватель**. Для каждой роли реализовать соответствующие экраны и функционал.

### Роли и экраны

#### Студент
1. **Главная страница** — актуальное расписание для этого студента.
2. **Электронный журнал** — таблица с пропусками и актуальными оценками по предметам и датам.
3. **Страница предмета** — оценки и успеваемость по лабораторным, контрольным, практическим работам, тестам/устным опросам.
4. **Страница лабораторной работы** — возможность прикрепить решение, видеть комментарии преподавателя, отметку, дату выдачи, дедлайн, напарников (если работа командная), теоретические материалы.

#### Преподаватель
1. **Главная страница** — актуальное расписание преподавателя.
2. **Журнал** — отображение всех предметов и групп, где ведёт преподаватель. Возможности:
   - Добавлять день (урок) в журнал
   - Выставлять оценки, опоздания (центральная кнопка мыши), отсутствия (правая кнопка мыши)
   - Оценка выставляется через всплывающее окно или с клавиатуры (валидация: только числа)
   - Автоматическое определение опозданий по времени начала/конца урока (тайминги хардкожены в БД)
   - Обесцвечивание строки отчисленного студента, выделение новых студентов
   - Подсветка клетки, строки и колонки при наведении мыши
3. **Программа по предмету** — шаблон занятий (лабы, теория, практика, контрольные), возможность выставлять дедлайны, комментарии, выкладывать ТЗ, настраивать команду студентов для командных работ
4. **Экран сдачи лабораторных** — список студентов, сдавших лабораторные, сроки сдачи, ссылки на файлы/решения, возможность проверить работу, выставить отметку и комментарий

## Технологии

- **Фронтенд:** React/Vue/Angular/Next.js (на выбор) + вспомогательные модули (например, Material UI, Bootstrap, Ant Design)
- **Бэкенд:** Node.js + TypeScript + Express или Nest.js
- **База данных:** обязательно использовать любую СУБД на выбор (например, PostgreSQL, MySQL, SQLite, MongoDB и др.)
- **Аутентификация:** JWT или сессии (cookie)
- **Файловое хранилище:** локально или через облако

## Требования к реализации

- Обязательное использование базы данных для хранения всех данных приложения (пользователи, расписание, оценки, лабораторные, файлы и т.д.)
- Чистая архитектура: разделение фронта и бэка, REST API
- Реализация ролей и разграничения доступа
- Валидация данных на сервере и клиенте
- Современный UI/UX (адаптивность, подсветка, всплывающие окна)
- Хранение и отображение файлов лабораторных работ
- Возможность комментирования работ преподавателем
- Поддержка командных лабораторных
- README с инструкцией по запуску

## Критерии оценки
- Полнота реализации всех экранов и ролей
- Корректность работы журналов, расписания, лабораторных
- Качество кода, структура проекта, оформление UI
- Реализация валидации, разграничения доступа
- README и инструкция по запуску

## Требования к команде
- Команда состоит из **не более** 5 человек
- В команде есть явный тим-лид, который будет нести всю ответственность несдачу/сдачу проекта
- Команда **может** состоять из 1 человека, но в таком случае опросы по коду будут существенно серьезнее
